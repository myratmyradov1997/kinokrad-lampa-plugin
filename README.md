# KinoKrad для Lampa

Отдельный плагин Lampa для каталога `kinokrad.my`. Работающий Dorama-плагин не изменяется.

Что реализовано:

- фильмы и сериалы из сортировки «Топ за неделю», первые три страницы каждого раздела;
- собственная подробная карточка: описание, постер, год, страна, жанры, рейтинги KinoKrad/КП/IMDb, актёры и режиссёр;
- выбор версии/озвучки фильма;
- выбор сезона, серии и озвучки сериала;
- адаптивный HLS-master со всеми доступными качествами;
- обновление истёкшей ссылки после ошибки плеера;
- безопасный media proxy с allowlist доменов;
- одинаковая активация мышью и кнопкой OK на пульте (`click` + `hover:enter`).

## Локальный запуск

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/playwright install chromium
PORT=5200 .venv/bin/python server.py
```

Проверка: `http://127.0.0.1:5200/api/health`.

Адрес плагина для Lampa: `http://IP_СЕРВЕРА:5200/plugin.js`.

## Почему backend использует Chromium

KinoKrad-плеер выдаёт HLS только с динамической подписью `Borth`. Её создаёт защитный браузерный слой, она меняется при каждом запуске. Backend открывает доверенный iframe в headless Chromium, перехватывает одноразовую подпись, получает выбранный файл и кеширует результат. Телевизор не выполняет эту защитную логику и получает обычный HLS.

## Тесты

```bash
pytest -q
node --check plugin.js
```

Сервис рассчитан на один Gunicorn worker: резолвер сериализован, а готовые потоки кешируются на 90 минут.

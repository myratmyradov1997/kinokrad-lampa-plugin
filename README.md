# KinoKrad для Lampa

Плагин просмотра KinoKrad для стандартных карточек и поиска Lampa. Работающий Dorama-плагин не изменяется.

Что реализовано:

- кнопка «Смотреть в KinoKrad» в обычной подробной карточке Lampa;
- отдельный источник KinoKrad в штатном поиске Lampa;
- описания, рейтинги и постеры показывает стандартная карточка Lampa/TMDB;
- поиск KinoKrad использует быстрый штатный AJAX сайта, а Chromium остаётся резервом при изменении защиты;
- карточка и player metadata обычно загружаются двумя быстрыми curl-запросами; единый Chromium-проход остаётся резервом;
- выбор версии/озвучки фильма;
- выбор сезона, серии и озвучки сериала;
- явный выбор всех доступных качеств в Lampa и запуск максимального качества по умолчанию;
- сохранение обязательных KinoKrad/VK CDN заголовков для master, сегментов и субтитров;
- обновление истёкшей ссылки после ошибки плеера;
- безопасный media proxy с allowlist доменов;
- автоматический Chromium fallback, если KinoKrad блокирует серверный HTTP защитным 403/404;
- штатная прокрутка `Lampa.Scroll` принудительно синхронизируется после стрелок пульта; три пути активации на TV: `hover:enter`, `Controller.enter` и мышь `click`.

## Локальный запуск

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/playwright install chromium
PORT=5200 .venv/bin/python server.py
```

Проверка: `http://127.0.0.1:5200/api/health`.

Адрес плагина для Lampa: `http://IP_СЕРВЕРА:5200/plugin.js`.

## Почему backend использует Chromium

KinoKrad-плеер выдаёт HLS только с динамической подписью `Borth`. Её создаёт защитный браузерный слой, она меняется при каждом запуске. Backend создаёт минимальный доверенный iframe-контекст в headless Chromium, даёт guard-слою подписать запрос строго выбранного файла и кеширует результат. Полная тяжёлая карточка KinoKrad при этом не загружается. Телевизор не выполняет защитную логику и получает обычный HLS.

## Тесты

```bash
pytest -q
node --check plugin.js
```

Сервис рассчитан на один Gunicorn worker: резолвер сериализован, а готовые потоки кешируются на 90 минут.

# KinoKrad для Lampa

Плагин просмотра KinoKrad для стандартных карточек и поиска Lampa. Работающий Dorama-плагин не изменяется.

Что реализовано:

- кнопка «Смотреть в KinoKrad» в обычной подробной карточке Lampa;
- отдельный источник KinoKrad в штатном поиске Lampa;
- описания, рейтинги и постеры показывает стандартная карточка Lampa/TMDB;
- поиск KinoKrad выполняется только по запросу пользователя и кешируется — каталог больше не задерживает запуск плагина;
- выбор версии/озвучки фильма;
- выбор сезона, серии и озвучки сериала;
- адаптивный HLS-master со всеми доступными качествами;
- сохранение обязательных KinoKrad/VK CDN заголовков для master, сегментов и субтитров;
- обновление истёкшей ссылки после ошибки плеера;
- безопасный media proxy с allowlist доменов;
- автоматический Chromium fallback, если KinoKrad блокирует серверный HTTP защитным 403/404;
- штатная прокрутка `Lampa.Scroll` и одинаковая активация мышью и кнопкой OK на пульте (`click` + `hover:enter`).

## Локальный запуск

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/playwright install chromium
PORT=5200 .venv/bin/python server.py
```

Проверка: `http://127.0.0.1:5200/api/health`.

Адрес плагина для Lampa: `http://IP_СЕРВЕРА:5200/plugin.js`.

## Почему backend использует Chromium

KinoKrad-плеер выдаёт HTML и HLS только после реального сценария «карточка → Смотреть» и с динамической подписью `Borth`. Её создаёт защитный браузерный слой, она меняется при каждом запуске. Backend воспроизводит этот сценарий в headless Chromium, получает `fileList`, даёт guard-слою подписать запрос выбранного файла и кеширует результат. Телевизор не выполняет эту защитную логику и получает обычный HLS.

## Тесты

```bash
pytest -q
node --check plugin.js
```

Сервис рассчитан на один Gunicorn worker: резолвер сериализован, а готовые потоки кешируются на 90 минут.
